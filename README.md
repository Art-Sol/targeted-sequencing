# Targeted Sequencing

Десктопное приложение для запуска пайплайна таргетного секвенирования (на основе Docker) и просмотра результатов. Предназначено для биологов и исследователей, работающих с FASTQ-данными — без необходимости разбираться в командной строке или Docker напрямую.

Работает **полностью offline** на Windows, macOS и Linux. Анализ выполняется локально в Docker-контейнере, результаты отображаются в интерактивной таблице с возможностью переключения между метриками (mapped reads / RPKM) и экспорта в CSV.

## Возможности

- Загрузка `list_reads.txt` и FASTQ-файлов (single-end и paired-end), drag-and-drop
- Запуск Docker-пайплайна с автоматическим управлением ресурсами — число потоков подбирается под систему
- Матрица результатов «образцы × таргеты» с подсветкой нулевых ячеек
- История всех запусков, переключение между ними и удаление лишних для освобождения места
- Экспорт в CSV (UTF-8 BOM + `;` для совместимости с русским Excel)
- Автоматическая загрузка bundled Docker-образа при первом запуске — интернет не требуется

## Технологии

React + TypeScript + Ant Design (Vite) / Node.js + Express / Electron

## Что нужно перед установкой

Приложение использует **Docker** для запуска самого пайплайна анализа. Docker нужно установить **отдельно**, до запуска приложения.

| ОС | Что устанавливать | Где скачать |
|----|-------------------|-------------|
| Windows | Docker Desktop | https://docs.docker.com/desktop/setup/install/windows-install/ |
| macOS | Docker Desktop | https://docs.docker.com/desktop/setup/install/mac-install/ |
| Linux | Docker Engine | https://docs.docker.com/engine/install/ |

После установки Docker — **запустите его** (на Windows/macOS — иконка Docker Desktop в системном трее должна стать «зелёной»; на Linux — `sudo systemctl start docker`).

> Если вы не уверены, установлен ли Docker — приложение само проверит при запуске и подскажет.

## Windows

1. Скачайте файл `targeted-sequencing-setup-0.1.0.exe`.
2. Запустите его двойным кликом.
3. **Если Windows покажет синее окно «Windows protected your PC»** — это нормально, наше приложение не подписано цифровым сертификатом (он стоит дорого, для внутреннего использования его не покупаем):
   - Нажмите **«More info»** (или «Подробнее»).
   - Внизу появится кнопка **«Run anyway»** (или «Выполнить в любом случае») — нажмите её.
4. Пройдите стандартный мастер установки: «Next» → «Install» → «Finish».
5. На рабочем столе появится ярлык **«Targeted Sequencing»** — двойной клик запускает приложение.

> **Куда устанавливается:** `C:\Users\<ваше имя>\AppData\Local\Programs\Targeted Sequencing\`. Прав администратора не требуется.

## macOS

1. Скачайте файл `targeted-sequencing-0.1.0-x64.dmg` (для Intel-Mac) или `targeted-sequencing-0.1.0-arm64.dmg` (для Apple Silicon, M1/M2/M3).
2. Откройте `.dmg` двойным кликом — появится окно с иконкой приложения и папкой Applications.
3. **Перетащите** иконку приложения в папку Applications.
4. **Первый запуск** — macOS может заблокировать приложение со словами **«cannot be opened because the developer cannot be verified»** (Gatekeeper). Это потому что приложение не подписано Apple-сертификатом. Есть два способа обойти:

   **Способ 1 — через системные настройки (проще):**
   - Откройте System Settings → Privacy & Security.
   - Прокрутите вниз — увидите блок «"Targeted Sequencing" was blocked from use because it is not from an identified developer».
   - Нажмите **«Open Anyway»**.
   - В появившемся диалоге снова нажмите **«Open»**.

   **Способ 2 — через терминал (надёжнее):**
   ```bash
   xattr -d com.apple.quarantine "/Applications/Targeted Sequencing.app"
   ```
   После этого приложение запускается обычным двойным кликом.

5. Дальнейшие запуски — обычным двойным кликом из папки Applications или Launchpad.

## Linux

Доступны два формата:

### AppImage (рекомендуется)

Универсальный формат, работает на любом Linux-дистрибутиве, не требует установки.

```bash
chmod +x targeted-sequencing-0.1.0.AppImage
./targeted-sequencing-0.1.0.AppImage
```

Можно перенести файл в любую удобную папку, ярлык создавать необязательно.

### deb (для Debian / Ubuntu / Mint и производных)

```bash
sudo dpkg -i targeted-sequencing-0.1.0.deb
```

После установки приложение появится в меню приложений как **Targeted Sequencing**.

> **Linux + Docker:** убедитесь, что ваш пользователь добавлен в группу `docker` (иначе приложение не сможет запускать контейнеры без `sudo`):
> ```bash
> sudo usermod -aG docker $USER
> ```
> После этого нужно **перелогиниться** (выйти и зайти в систему).

## Первый запуск

После запуска приложения произойдёт следующее:

1. Откроется главное окно приложения.
2. Приложение проверит, установлен ли Docker и запущен ли он. Если что-то не так — на экране будет понятная инструкция.
3. Если Docker есть, но образа пайплайна (~30 МБ) ещё нет в системе — приложение **автоматически** загрузит его из встроенной копии. Это занимает 30-60 секунд при первом запуске; статус виден на экране.
4. После загрузки образа интерфейс становится доступным. Можно загружать файлы и запускать анализ.

## Если что-то пошло не так

### Приложение не запускается

Найдите файл лога — он автоматически создаётся при каждом запуске:

| ОС | Путь к лог-файлу |
|----|------------------|
| Windows | `C:\Users\<ваше имя>\AppData\Roaming\targeted-sequencing\logs\main.log` |
| macOS | `~/Library/Application Support/targeted-sequencing/logs/main.log` |
| Linux | `~/.config/targeted-sequencing/logs/main.log` |

> На Windows папка `AppData` скрытая. Самый быстрый способ открыть нужную папку: нажать `Win+R`, ввести `%APPDATA%\targeted-sequencing\logs` и нажать Enter.

### Анализ запускается, но падает с ошибкой

В этом же файле `main.log` будут детали ошибки от Docker'а. Также можно посмотреть лог самого пайплайна:

| ОС | Путь к логу пайплайна |
|----|-----------------------|
| Windows | `%APPDATA%\targeted-sequencing\pipeline-workdir\logs\pipeline_<дата>.log` |
| macOS | `~/Library/Application Support/targeted-sequencing/pipeline-workdir/logs/pipeline_<дата>.log` |
| Linux | `~/.config/targeted-sequencing/pipeline-workdir/logs/pipeline_<дата>.log` |

### Закончилось место на диске

Все рабочие данные (входные FASTQ-файлы, результаты прошлых запусков) лежат в той же папке `pipeline-workdir`. Размер может быть большим — каждый FASTQ-файл может весить несколько ГБ.

В самом приложении на странице **«Выполненные анализы»** есть кнопки удаления отдельных запусков и «Удалить все» — пользуйтесь ими для освобождения места.

### Не помогло — что прислать разработчику

Скопируйте файл `main.log` (см. путь выше) и пришлите. Этого почти всегда достаточно для диагностики.

## Удаление приложения

| ОС | Как удалить |
|----|-------------|
| Windows | «Параметры» → «Приложения» → найти «Targeted Sequencing» → «Удалить» |
| macOS | Перетащить `Targeted Sequencing.app` из папки Applications в Корзину |
| Linux (AppImage) | Просто удалить `.AppImage`-файл |
| Linux (deb) | `sudo apt remove targeted-sequencing` |

> При удалении **пользовательские данные** (`pipeline-workdir/`, лог-файлы) **сохраняются** — это специально, чтобы при переустановке вы не потеряли результаты прошлых анализов. Если хотите удалить вообще всё — удалите вручную папку `targeted-sequencing` из соответствующего пути (см. таблицу «лог-файла» выше).

## Сборка из исходников (для разработчика)

Этот раздел нужен только тем, кто склонировал репозиторий и хочет собрать установщик самостоятельно. Конечному пользователю он не нужен — достаточно скачать готовый файл и установить по инструкции выше.

### Системные зависимости

| Что | Версия | Где взять |
|-----|--------|-----------|
| **Node.js** | 20 LTS | https://nodejs.org/ или через NodeSource |
| **Docker** | любая актуальная | Docker Desktop (Win/Mac) или Docker Engine (Linux) |
| **libfuse2** | — | только Linux: `sudo apt install libfuse2` (нужно для AppImage) |

### Получение Docker-образа пайплайна

Файл `electron/resources/targets-pipeline_0.1.0.tar.gz` (~30 МБ) **в репозитории отсутствует** — он в `.gitignore` из-за размера. Разработчик готовит его сам.

Если у вас есть доступ к Docker-образу `targets-pipeline:0.1.0` (через `docker pull` из приватного registry или предоставленный мейнтейнером проекта tar):

```bash
docker pull targets-pipeline:0.1.0
docker save targets-pipeline:0.1.0 | gzip > electron/resources/targets-pipeline_0.1.0.tar.gz
```

Если образа нет в наличии — обратитесь к мейнтейнеру проекта за готовым `tar.gz`-файлом и положите его по тому же пути.

Без этого файла установщик соберётся, но при первом запуске приложение не сможет автоматически загрузить образ — пользователю придётся ставить его вручную через `docker pull`.

### Установка зависимостей и запуск в dev-режиме

```bash
npm install
npm run dev
```

`npm run dev` поднимает Vite-dev-server и Express-бэкенд параллельно. Открывайте http://localhost:5173.

Для запуска в Electron-обёртке (с локальным API-сервером и проверками Docker):
```bash
npm run electron:dev
```

### Сборка production-инсталлятора

Команды зависят от целевой ОС — кросс-сборка из-под одной ОС в другую не поддерживается (AppImage требует Linux, DMG — macOS, NSIS — Windows):

```bash
npm run dist:win      # на Windows → release/targeted-sequencing-setup-0.1.0.exe
npm run dist:mac      # на macOS  → release/*.dmg (x64 + arm64)
npm run dist:linux    # на Linux  → release/*.AppImage и *.deb
```

Готовые артефакты появятся в папке `release/`.
